Systems Analysis and Database Design

This project spanned over the course of two semester of my junior year of college. The purpose of the project was to analyze Tista's Fine Furniture's core business process and create a system that will not only satisfy the client's needs, but also reduce the costs and improve efficiency.

Systems Analysis

During the first semester, my project team, jSack-Tech, conducted a systems analysis on the business. We completed various project deliverables that helped analyze the client's current business process, the impact of their problems, and solutions to overcome them. In order to understand the client's needs, we formulated four critical requirements to provide a base for the core functionality of the database application: manage sale orders, manage customers, manage vendors, and reduce running costs. The solution that the client chose was to create a Java based application backed by a SQL Server database.

Database Design

After gathering and analyzing the business process and requirements for Tista's application, we designed an entity relationship diagram that served as a blueprint for our database development. The final ERD was composed of 41 tables with 6 of those identifying as a core table for the application's functionality. A data dictionary was also created for the purpose of defining each table and attribute in the database. To test the database, hundreds of rows of test data was inserted into the tables. The database was successfully tested by running SQL scripts that were created to reflect the various scenarios of the business.
